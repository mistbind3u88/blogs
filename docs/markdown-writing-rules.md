# Markdown と文章ルール

このリポジトリの Markdown は、エディタ内で完結する記法としてだけでなく、外部の投稿先へ転記する前提でも扱う。整形のしやすさより、転記後の読みやすさを優先する。

## 基本の書き方

- 見出し、段落、空行の構造を保つ。
- 段落は原則として 1 段落 1 行で書く。
- 文途中の改行は避ける。
- 見出しは `#` から `####` までを基本とし、`#####` 以上は原則使わない。

## こうしている理由

主な公開先のひとつである `note` は WYSIWYG 編集が前提で、Markdown 上の途中改行がそのまま表示に影響しやすい。エディタ都合の折り返しを本文へ持ち込まないため、ハードラップではなく段落単位で文章を保つ。

## 強調と埋め込み

- `Bold` は使ってよい。
- 括弧を含む強調では、エディタ上の見え方と投稿先での見え方がずれることがあるため、見た目の確認を意識する。
- `note` では埋め込み系のマークアップに頼らず、必要な外部サービスの URL は投稿先側で扱う前提にする。

## ルビ

ルビは投稿先ごとの差が大きい。

- カクヨムで漢字に自然にルビを振るだけなら `漢字《かな》` 形式で足りることがある。
- `note` では `|範囲《かな》` のように、ルビを振る範囲を明示する前提で書く。
- ひらがなや複数語にまたがるルビも、範囲を明示する書き方を優先する。

## 文章上の判断基準

- Markdown 記法の都合で文章を不自然にしない。
- 読者に見せる本文と、作業用のメモや参考資料は見出しで区切って混在を明示する。
- 執筆中のメモを残す場合でも、後で本文と誤認しにくい構造にしておく。
