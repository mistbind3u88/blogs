# 収録文書の案内

このディレクトリには、リポジトリ全体で繰り返し参照する判断基準を置く。設定ファイルやテンプレートを読むだけでは分かりにくい運用意図、配置判断、文書化方針をここで補う。

## この文書の役割

- トップレベル `AGENTS.md` から参照する恒久文書の一覧を示す。
- 各文書が何の判断に使うものかを短く説明する。
- ここに追加してよい文書と、別の場所に置くべき文書を切り分ける。

## ここに置く内容

- 複数カテゴリにまたがって使う判断基準
- エージェントや保守者が配置、記述、運用で迷ったときの判断材料
- 設定ファイルの値そのものではなく、その背景にある運用意図
- 単発の作業メモではなく、継続して参照する前提の説明

## 収録中の文書

- `repository-philosophy.md`
  - このリポジトリで何を優先し、どの観点で判断するかをまとめる。
- `content-structure.md`
  - どのトップレベルディレクトリに何を置くか、迷いやすい配置判断をまとめる。
- `markdown-writing-rules.md`
  - Markdown の書き方、媒体差、本文ファイルと作業メモの分離方針をまとめる。
- `github-workflow.md`
  - Issue、PR、Milestone、ブランチ、コミットに関するリポジトリ共通の運用をまとめる。

## ここに置かない内容

- `.editorconfig`、`.gitattributes`、`.prettierrc.yaml` を見れば足りる整形規約
- `.markdownlint.jsonc` や `.textlintrc.json` の設定値一覧
- `.github/workflows/*.yml` の実行条件やコマンドの写し
- 特定カテゴリだけに閉じた執筆ルール
- 特定 skill の手順説明や、一時的な作業メモ
