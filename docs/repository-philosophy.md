# リポジトリの基本方針

このリポジトリは、個人で書くブログ記事やその周辺メモを履歴管理するための場である。完成原稿だけを置くのではなく、草案、計画、振り返り、運用改善の記録も含めて扱う。

## 何を残すか

- 記事本文そのもの
- 公開前の草案や構成メモ
- 旅行や創作のように、後で本文へ育てるための下書き
- 記事づくりを支える運用ルールやサマリ生成手順

## 何を優先するか

- 文章の完成度だけでなく、後で読み返して再利用できることを重視する。
- ひとつの題材について、公開前後の試行錯誤や判断の跡が追える状態を保つ。
- リポジトリ全体の運用改善も、記事執筆と同じく継続対象として扱う。

## エージェントが守るべき姿勢

- 既存の文体やトーンを不用意に均質化しない。
- 人が後で読み返して判断できるよう、変更理由が伝わる単位で作業する。
- 記事本文とリポジトリ運用の改善が同時に必要でも、目的が異なる変更は分けて扱うことを優先する。
- AI 支援を使う場合でも、本文、参考メモ、生成支援の痕跡を混同しない。
