# 活動サマリ

このリポジトリの活動サマリは、Issue / PR の件数を並べるためのものではなく、その期間に何を書き、何を整え、どんな流れで動いていたかを一般の読み手にも分かる形で残すためのものとする。

## 対象と出力先

- 年次サマリは `summaries/YYYY.md`
- 半期サマリは `summaries/half-year/YYYY-H1.md` と `YYYY-H2.md`

半期サマリはファイル名だけで期間が分かるよう、`half-year` 直下に置く。

## 集計の前提

- Issue は `closedAt` を基準に集計する。
- PR は `mergedAt` を基準に集計する。
- `created` 基準と混在させない。

カテゴリ整理は、Issue / PR タイトルに含まれるラベル風の文字列をもとに行う。

## 文章の方針

- 本文と見出しは日本語で書く。
- Git や実装寄りの用語を、そのまま見出しや説明へ持ち込まない。
- 一般の読み手が分かる語彙へ言い換える。
- 件数の言い換えだけで終わらせず、題材と作業内容の具体を入れる。

特に `なにやってたの？` に相当する説明では、何の題材に取り組んでいたか、その題材で何をしていたか、書くだけでなく整備や振り返りまで含めてどう動いていたかを具体例付きで書く。

## 生成の考え方

集計はスクリプトで行い、文章はエージェントが書く。集計や再生成を行う場合は `skills/summaries-periodic-workflow/SKILL.md` を使う。

変更内容から見える作業の実体も本文に反映する。複数ファイルにまたがる更新や、README / AGENTS / SKILL を同時に触っているケースは、単なる件数ではなく期間の手触りとして要約する。

## 時点表記

サマリには必ず集計時点を具体的な日付で書く。相対表現ではなく、`2026-03-12 JST 時点` のように明記する。
