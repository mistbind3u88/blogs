# コンテンツ構造

トップレベルのディレクトリは、記事の形式ではなく題材や用途で分ける。設定ファイルはルートに置き、本文やメモは下位ディレクトリへ寄せる。

## トップレベルの見方

- `entertainments`
  - 漫画、映画、競馬予想など、エンタメ作品や鑑賞対象に寄った記録を置く。
- `games`
  - プレイ記録だけでなく、ゲームデザインや攻略の考察も含める。
- `stories`
  - 三題噺などの創作を置く。
- `journeies`
  - 旅行記、旅行計画、旅行メモを置く。
- `essays`
  - 生活や考え方を主題にした文章を置く。
- `carrier`
  - キャリア整理や面談向けの文章を置く。
- `summaries`
  - Issue / PR ベースで活動を振り返るサマリを置く。
- `skills`
  - Codex に渡す再利用可能な作業手順を置く。

## 配置判断の基準

- まず「何について書くか」で置き場所を決める。
- 同じ題材で、草案と完成版相当の文書が混在してもよい。用途より題材のまとまりを優先する。
- 運用説明やワークフローは、記事本文の近くではなく `docs` や `skills` に寄せる。
- 作業手順として再利用するものは `skills`、リポジトリ固有の方針説明は `docs` に置く。

## 投稿先との関係

このリポジトリは単一媒体専用ではない。現在の主な投稿先は `note`、`カクヨム`、`はてなブログ` で、同じ題材でも媒体ごとに見せ方や記法上の制約が異なる。そのため、本文ファイルは公開先そのものよりも、題材や作業途中の文脈を軸に整理する。
